# Agent Guidance

Use this file when editing repository documentation, especially Markdown rendered on GitHub.

---

## Documentation style

Write all repository documentation in the Swiss Design for GitHub Markdown style from [`mrjf/swiss-design-skill-md`](https://github.com/mrjf/swiss-design-skill-md).

Apply the style through native Markdown only.

| Principle | Repository expectation |
| --- | --- |
| Structure is the grid | Use clear heading hierarchy, tables for real tabular data, and ordered section flow |
| Whitespace is structure | Keep paragraphs short and separate major sections deliberately |
| Hierarchy is earned | Prefer concise labels, sequence, and placement over decorative emphasis |
| One accent idea | Use a single repeated marker system when useful, such as `01`, `02`, `03` |
| No hacks | Avoid CSS, scripts, custom fonts, layout tables, invisible characters, and decorative HTML |

---

## Pre-ship check

Before committing documentation changes:

- Confirm the first screen explains what the document is and what to do next.
- Keep heading levels sequential and meaningful.
- Use bold, italics, and inline code sparingly.
- Avoid emoji, ASCII banners, and decorative badge clusters.
- Ensure the Markdown remains readable as plain source.
